## 技术决策记录

1. **OCR选型**: 选 PaddleOCR 而非 Tesseract
   - 理由：日文识别准确率更高
   - 代价：首次使用需下载模型

2. **爬虫默认**: requests 而非 Playwright
   - 理由：更快、更轻量
   - 例外：动态加载页面/JS渲染

3. **数据库**: SQLite
   - 理由：零配置、文件存储、易备份

4. **前后端**: 不分离
   - 理由：用户要求"打开网页即可用"
   - 简化部署流程

## 注意事项

1. **依赖安装**:
   ```bash
   pip install -r requirements.txt
   playwright install chromium
   ```

2. **首次OCR使用**:
   - PaddleOCR 会自动下载日文模型（约100MB）
   - 需要联网

3. **PDF OCR依赖**:
   - 需要系统安装 poppler（pdf2image 依赖）
   - Windows: 下载 poppler 并配置 PATH
